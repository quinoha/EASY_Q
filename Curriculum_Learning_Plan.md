# 커리큘럼 학습(Curriculum Learning) 구현 계획

이 문서는 논문에서 제시된 **에러율(p-value) 기반 커리큘럼 학습**을 현재 시스템에 도입하기 위한 작업 계획서입니다.
(타겟 하드웨어: RTX 6000 Ada / A100 수준의 고성능 GPU 기준)

## 1. 논문 기준 스펙 요약
- **전체 학습 스텝 (Total Steps)**: 80,000 steps
- **배치 사이즈 (Batch Size)**: 3,328
- **학습 스케줄 (Curriculum Schedule)**:
  - **초반 2% (1,600 steps)**: 아주 낮은 `p_start` 값에서 시작하여 목표 에러율인 `p_target`까지 선형적으로 증가 (Linear Annealing).
  - **나머지 98% (78,400 steps)**: `p_target`으로 에러율을 고정하고 모델이 완전히 수렴할 때까지 학습 (Training at Target).

---

## 2. 해결해야 할 과제 및 고성능 GPU를 위한 구현 방안

### A. 동적 에러율 스케줄러 (Dynamic p-rate Scheduler)
- 현재 `train.py`는 학습을 시작할 때 고정된 `p_rate` 하나만 사용합니다.
- 루프 내에서 현재 `step`을 기반으로 `current_p`를 계산하는 로직을 추가해야 합니다.
  ```python
  warmup_steps = int(total_steps * 0.02)
  if step < warmup_steps:
      current_p = p_start + (p_target - p_start) * (step / warmup_steps)
  else:
      current_p = p_target
  ```

### B. On-the-fly 데이터 생성기(Stim Sampler) 업데이트 비용 최적화
- 초반 2% 구간에서는 매 스텝마다 에러율이 변하므로, Stim Circuit을 새로 구성하고 Sampler를 재컴파일해야 합니다.
- **최적화 전략**: 
  - `p`값이 변하는 첫 2% 구간에서는 매 스텝 재컴파일을 수행합니다. (A100급 스펙에서는 연산 코어와 통신 병목만 없다면 충분히 감당할 수 있는 오버헤드입니다.)
  - 2% 이후부터는 `p`값이 `p_target`으로 고정되므로, 샘플러를 한 번만 컴파일한 뒤 캐싱(Caching)하여 재사용함으로써 GPU 연산량을 최대치로 뽑아냅니다.

### C. 고성능 GPU 메모리 활용 (배치 사이즈 최적화)
- RTX 6000 Ada(48GB)나 A100(40GB/80GB)과 같은 대용량 VRAM 환경에서는 복잡하게 쪼개서 학습(Gradient Accumulation)할 필요 없이 **논문에서 제시된 배치 사이즈 `3,328`을 한 번에 메모리에 올릴 수 있을 확률이 높습니다.**
- 복잡한 누적 로직은 제거하고 바로 `batch_size=3328`을 주입하여 GPU 메모리를 최대한 활용하고, 연산 속도를 극대화하는 직관적인 방향으로 구현합니다. (만약 Distance가 커져서 일시적인 OOM이 발생할 경우에만 대비하여 배치 사이즈 조절 옵션만 열어둡니다.)

## 3. 코드 수정 작업 리스트 (train.py)

- [ ] **1단계: 파라미터 재구성**
  - `train_model` 함수 인자에 `p_start`, `p_target`, `total_steps=80000`, `batch_size=3328` 추가.
- [ ] **2단계: 학습 루프 내 스케줄링 및 샘플러 갱신 로직 추가**
  - 스텝별 `current_p` 계산.
  - `current_p`가 직전 스텝과 달라졌을 때만 Stim Sampler를 재컴파일.
- [ ] **3단계: 메모리 할당 및 GPU 활용 로직 확인**
  - 3,328개의 큰 배치를 `stim`으로 실시간 런타임 생성할 때 병목이 없는지 확인하고, PyTorch 텐서 변환 및 `to(device)` 로직 점검.
- [ ] **4단계: 로깅 강화**
  - `tqdm` 진행 바에 현재 `p` 값을 함께 출력하여 스케줄링이 올바르게 되고 있는지 모니터링 기능 강화.

---

## 4. 기대 효과
이러한 점진적 학습을 도입하면, 초반의 낮은 오류율에서 모델이 '신드롬의 기본 패턴'을 빠르게 암기하고, 점차 노이즈를 섞어가며 강건한(Robust) 디코더로 수렴하는 완벽한 흐름을 만들 수 있습니다. 빵빵한 GPU 스펙에 맞춰 데이터를 한꺼번에 쏟아부으므로 80,000 스텝의 긴 학습도 매우 빠르고 안정적으로 끝낼 수 있을 것입니다.
